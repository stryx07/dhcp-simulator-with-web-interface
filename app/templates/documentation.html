Understanding DHCP Vulnerabilities

DHCP Protocol Basics

Dynamic Host Configuration Protocol (DHCP) is used to assign IP addresses and network configuration to devices automatically. The process typically follows the DORA exchange:

  • Discover: Client broadcasts looking for a server.
  • Offer: Server offers an IP address.
  • Request: Client requests the offered IP.
  • Ack: Server acknowledges and reserves the IP.

Attack Scenarios

1. DHCP Starvation (Saturation)

An attacker floods the DHCP server with DISCOVER packets using spoofed MAC addresses. This exhausts the server's IP address pool, preventing legitimate clients from connecting to the network.

2. IP Conflict

An attacker statically assigns themselves an IP that is already in use or currently being offered by the DHCP server, causing network instability for the legitimate owner of that IP.

3. DHCP Spoofing (Rogue Server)

An attacker sets up a rogue DHCP server to answer DISCOVER requests faster than the legitimate server, assigning clients malicious DNS servers or gateways (Man-in-the-Middle).
